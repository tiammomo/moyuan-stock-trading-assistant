# Dashboard Redesign

这个技能偏“产品后台视觉改版”。目标是把原本平铺直叙的页面重做成更高级、更稳定、更易扫读的 dashboard/workbench。

## 先读什么

- `references/dashboard-patterns.md`

## 设计流程

1. 先定义页面主任务
   - 监控
   - 比较
   - 执行
   - 排错
2. 识别当前页面的问题
   - 信息挤在一起
   - 没有焦点
   - 面板关系不清
   - 颜色与层级混乱
3. 先重构布局，再精修视觉
4. 需要时可先做静态 dashboard prototype，再回接真实数据

## 推荐做法

- 先画出 summary 带
- 再安排主图表/主列表/主结果流
- 辅助信息放二级面板或右栏
- 状态色、标签、图标和数字格式要统一
- sidebar、toolbar、panel header 要形成一套节奏

## 视觉方向

- 产品后台优先“干净、可信、专业”
- 允许个性，但不能牺牲数据可读性
- 仪表盘不是海报，装饰必须服从任务

## 特别适合这个仓库

- 左侧会话 + 中间消息流 + 右侧结果的三栏工作台
- watchlist 和模板管理页面
- 结果表格、卡片和对比视图
