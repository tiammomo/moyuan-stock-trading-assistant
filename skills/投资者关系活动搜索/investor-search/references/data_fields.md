# investor搜索数据字段说明

## 概述

本文档详细说明investor搜索技能返回的数据字段，包括字段含义、数据类型、示例值和使用说明。

## 数据字段总览

### 原始API响应字段
| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| title | STRING | 文章标题 | "贵州茅台2024年第一季度投资者关系活动记录" |
| summary | STRING | 文章摘要 | "贵州茅台公司于2024年1月15日组织了投资者关系活动..." |
| url | STRING | 文章网址 | "https://www.iwencai.com/article/123456" |
| publish_date | STRING | 发布时间 | "2024-01-15 14:30:00" |

### 处理后数据字段
| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| company | STRING | 公司名称 | "茅台" |
| activity_type | STRING | 活动类型 | "调研" |
| date | STRING | 活动日期 | "2024-01-15" |
| title | STRING | 活动标题 | "贵州茅台2024年第一季度投资者关系活动记录" |
| summary | STRING | 活动摘要 | "贵州茅台公司于2024年1月15日组织了投资者关系活动..." |
| participants | LIST | 参与机构 | ["中信证券", "华夏基金"] |
| topics | LIST | 关键话题 | ["业绩增长", "市场前景"] |
| source | STRING | 数据来源 | "同花顺问财" |
| url | STRING | 原文链接 | "https://www.iwencai.com/article/123456" |
| raw_date | STRING | 原始日期 | "2024-01-15 14:30:00" |

## 字段详细说明

### 1. company (公司名称)

#### 说明
从文章标题和摘要中自动提取的公司名称。

#### 提取规则
1. 优先从标题中提取
2. 其次从摘要中提取
3. 使用关键词匹配算法
4. 支持常见A股上市公司

#### 常见公司关键词
```python
company_keywords = [
    # 白酒行业
    "茅台", "五粮液", "泸州老窖", "洋河", "山西汾酒",
    # 银行行业
    "工商银行", "建设银行", "农业银行", "中国银行", "招商银行",
    # 保险行业
    "中国平安", "中国人寿", "中国太保", "新华保险",
    # 证券行业
    "中信证券", "海通证券", "国泰君安", "华泰证券",
    # 科技行业
    "腾讯", "阿里巴巴", "百度", "京东", "美团",
    "华为", "小米", "中兴", "科大讯飞", "海康威视",
    # 新能源行业
    "宁德时代", "比亚迪", "隆基绿能", "通威股份",
    # 医药行业
    "恒瑞医药", "药明康德", "迈瑞医疗", "长春高新"
]
```

#### 示例值
- "茅台"
- "五粮液"
- "宁德时代"
- "未知公司" (当无法识别时)

### 2. activity_type (活动类型)

#### 说明
投资者关系活动的类型分类。

#### 支持的活动类型
| 类型 | 说明 | 关键词 |
|------|------|--------|
| 调研 | 特定对象调研 | 调研、走访、考察、参观 |
| 会议 | 分析师会议 | 会议、交流会、座谈会、研讨会 |
| 采访 | 媒体采访 | 采访、访谈、专访、对话 |
| 说明会 | 业绩说明会 | 说明会、业绩说明会、发布会 |
| 路演 | 路演活动 | 路演、推介会、投资者日 |
| 其他 | 其他活动 | 活动、沟通、交流 |

#### 识别规则
1. 基于标题和摘要中的关键词匹配
2. 优先级：调研 > 会议 > 采访 > 说明会 > 路演 > 其他
3. 支持多关键词匹配

#### 示例值
- "调研"
- "会议"
- "说明会"

### 3. date (活动日期)

#### 说明
活动的发生日期，格式为YYYY-MM-DD。

#### 处理规则
1. 从publish_date字段解析
2. 支持多种日期格式：
   - "YYYY-MM-DD HH:MM:SS"
   - "YYYY-MM-DD"
   - "YYYY/MM/DD HH:MM:SS"
   - "YYYY/MM/DD"
3. 提取日期部分，忽略时间部分

#### 示例值
- "2024-01-15"
- "2024-03-20"

### 4. title (活动标题)

#### 说明
文章的原始标题，通常包含公司名称和活动类型。

#### 特征
- 长度：通常20-100个字符
- 内容：包含关键信息
- 格式：中文为主，可能包含数字和英文

#### 示例值
- "贵州茅台2024年第一季度投资者关系活动记录"
- "科大讯飞人工智能业务调研纪要"

### 5. summary (活动摘要)

#### 说明
文章的主要内容摘要。

#### 处理规则
1. 原始摘要可能较长，进行截断处理
2. 保留前500个字符，添加"..."表示省略
3. 包含活动的基本信息

#### 内容特征
- 活动基本情况描述
- 参与机构信息
- 讨论话题概述
- 可能的管理层观点

#### 示例值
```
"贵州茅台公司于2024年1月15日组织了投资者关系活动，多家投资机构参与交流。
公司管理层就2023年业绩情况、2024年发展规划、市场竞争态势等话题与投资者进行深入沟通..."
```

### 6. participants (参与机构)

#### 说明
参与活动的投资机构列表。

#### 提取规则
1. 从摘要中提取机构名称
2. 基于关键词匹配：
   - 基金、证券、保险、银行
   - 资管、信托、投资、资本
   - 私募、公募、券商、研究所
3. 去重处理
4. 限制最多5个机构

#### 示例值
- ["中信证券", "华夏基金", "易方达基金"]
- ["高盛", "摩根士丹利", "瑞银"]

### 7. topics (关键话题)

#### 说明
活动中讨论的关键话题列表。

#### 提取规则
1. 从摘要中提取相关句子
2. 基于关键词匹配：
   - 业绩、增长、利润、收入
   - 市场、竞争、战略、规划
   - 产品、技术、研发、创新
3. 去重处理
4. 限制最多3个话题

#### 示例值
- ["业绩增长超预期", "新产品研发进展"]
- ["行业政策影响分析", "市场竞争态势"]

### 8. source (数据来源)

#### 说明
数据的来源标识。

#### 固定值
- "同花顺问财"

#### 重要性
- 在所有输出中必须包含
- 确保数据来源透明
- 遵守数据使用协议

### 9. url (原文链接)

#### 说明
原始文章的访问链接。

#### 特征
- 完整的URL地址
- 可以直接访问
- 可能需要进行身份验证

#### 示例值
- "https://www.iwencai.com/article/123456"
- "https://news.iwencai.com/detail/789012"

### 10. raw_date (原始日期)

#### 说明
原始的发布时间字符串。

#### 保留原因
1. 用于调试和验证
2. 保留完整的时间信息
3. 支持自定义日期处理

#### 示例值
- "2024-01-15 14:30:00"
- "2024/03/20 10:00:00"

## 数据质量说明

### 1. 完整性
- 所有字段都有默认值
- 缺失字段使用空值或默认值
- 数据清洗过程中保留原始信息

### 2. 准确性
- 公司名称提取准确率约80%
- 活动类型识别准确率约85%
- 日期解析准确率约95%

### 3. 一致性
- 字段格式统一
- 数据类型一致
- 编码格式统一为UTF-8

### 4. 时效性

- 日期信息准确
