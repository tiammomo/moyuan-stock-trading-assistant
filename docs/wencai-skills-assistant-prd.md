# 问财 Skills 个人助手 PRD

## 1. 背景

你已经装好了问财相关 skills，但当前实际使用仍然偏“手工”：

- 需要自己判断该问哪个 skill。
- 同一个问题往往要拆成多轮提问。
- 回答结果有时只有原始表格，没有形成适合个人投资决策的视图。
- 短线、波段、中线价值三种风格切换时，没有统一的用户画像和提问模板。
- 追问链路不顺，容易出现“上一轮说过的条件，这一轮又要重新说”的问题。

本项目的目标不是重新做一套问财，而是做一层更贴合你个人习惯的“问财 Skills 助手壳”。

## 2. 产品目标

### 2.1 总目标

做一个个人使用的 Web 助手，帮助你：

- 更顺滑地向问财 skills 发问。
- 自动选择或推荐合适的 skill。
- 把原始查询结果整理成更适合投资决策的结构化视图。
- 保留你的交易风格、仓位约束、偏好板块、常用追问方式。

### 2.2 目标边界

这个产品重点解决：

- 提问入口统一
- Skill 路由与编排
- 查询结果结构化
- 追问与比较
- 个人偏好记忆

这个产品暂不重点解决：

- 真正的实盘交易执行
- 高频量化回测系统
- 完整投研数据库建设
- 大团队权限系统

## 3. 目标用户

### 3.1 核心用户

你本人，具备如下特征：

- 主要关注 A 股
- 有短线、波段、中线价值三种不同操作场景
- 已经接受用 AI 和问财 skills 辅助决策
- 更重视“少折腾、快得到结构化结果”

### 3.2 后续可扩展用户

- 1 到 3 人的小型投研协作场景
- 自己的多个终端同步使用

## 4. 核心问题

当前体验中的关键问题：

1. 入口割裂：你知道装了很多 skill，但不知道每次该先用哪个。
2. 结果松散：问财给了数据，但没有直接变成“候选池、比较表、行动建议”。
3. 追问成本高：经常要重复“资金规模、持股周期、风险偏好、筛选条件”。
4. 上下文弱：系统不知道你是在做短线、波段还是中线价值。
5. 缺少解释：你会想知道“为什么这次用了这个 skill，而不是另一个”。

## 5. 产品定位

一句话定位：

“一个围绕问财 skills 的个人投资问答工作台，负责路由、整形、追问和偏好记忆。”

核心价值不是替代问财，而是做问财上层的个人操作系统。

## 6. 成功标准

### 6.1 用户体验目标

- 用户在 1 个输入框内完成大多数提问，不需要自己先判断 skill。
- 一次查询后可以自然进入“比较 / 过滤 / 继续追问 / 保存候选”的链路。
- 用户画像只需设置一次，后续默认带入。

### 6.2 可量化指标

- 80% 以上的查询不需要用户手动指定 skill。
- 70% 以上的结果能在首屏直接形成表格、卡片或比较视图。
- 追问时重复输入个人参数的次数下降 70%。
- 常见问句首屏响应：
  - 有缓存：3 到 5 秒
  - 无缓存：6 到 12 秒

## 7. 产品原则

1. 少让用户思考“该用哪个 skill”。
2. 每次结果都要给出“下一步能做什么”。
3. 明确区分“事实数据”和“系统判断”。
4. 结果优先结构化，不优先长篇文字。
5. 对个人投资风格要有状态感，不把每一轮都当新会话。

## 8. 典型使用场景

### 8.1 场景一：短线盘前找候选

输入：

`今天适合做什么方向？给我 5 只短线观察股`

系统应自动：

- 识别为短线模式
- 优先使用 `问财选A股`、`问财选板块`、`行情数据查询`、`事件数据查询`、`公告搜索`、`新闻搜索`
- 输出板块热度 + 候选股表 + 风险点 + 追问入口

### 8.2 场景二：波段挖掘

输入：

`找 5 只未来 2 到 4 周值得跟踪的趋势股`

系统应自动：

- 识别为波段模式
- 结合行业、行情、财务、事件、机构观点
- 输出候选表，并给出趋势状态、催化、关注区间、失效条件

### 8.3 场景三：中线价值研究

输入：

`帮我筛几只估值不贵、财务质量还不错的 A 股`

系统应自动：

- 识别为中线价值模式
- 优先使用 `财务数据查询`、`行业数据查询`、`公司股东股本查询`、`机构研究与评级查询`、`公告搜索`
- 输出中线研究列表与进一步研究建议

### 8.4 场景四：二次追问

输入：

`把刚才那 3 只股票做个对比`

系统应自动：

- 读取上一轮结果
- 识别“指代对象”
- 输出比较表而不是重新从头搜索

## 9. MVP 范围

## 9.1 必须有

- 单一问答入口
- 自动模式识别：短线 / 波段 / 中线价值 / 通用数据查询
- Skill 自动路由
- 多 skill 串联
- 结果结构化展示
- 追问建议
- 用户画像配置
- 查询历史
- 收藏候选股 / 保存会话

### 9.2 建议有

- 模板中心
- 比较模式
- 自定义输出格式
- 查询来源说明

### 9.3 可以后置

- 模拟炒股联动
- 自选股同步
- 预警推送
- 组合管理
- 多账户系统

## 10. 信息架构

建议产品结构：

1. 首页 / 主工作台
2. 历史会话
3. 模板中心
4. 收藏夹 / 候选池
5. 设置页

### 10.1 首页 / 主工作台

由 4 个区域构成：

- 顶部：模式切换、市场日期、快速模板
- 左侧：历史会话 / 收藏查询
- 中间：聊天与结果区
- 右侧：本轮命中的 skill、参数、候选股、追问建议

### 10.2 模板中心

包含：

- 短线模板
- 波段模板
- 中线价值模板
- 自定义模板

### 10.3 设置页

包含：

- 用户交易画像
- 默认持仓周期
- 单票仓位上限
- 最大回撤容忍
- 偏好行业
- 默认返回数量
- 是否自动展示“风险提示”

## 11. 核心功能设计

## 11.1 智能输入框

输入框支持：

- 自然语言提问
- 快捷模板插入
- 参数 chips
- 最近问句联想
- 追问上下文继承

输入框下方建议有快捷按钮：

- `短线`
- `波段`
- `中线价值`
- `比较刚才结果`
- `继续深挖`
- `换一批`

## 11.2 Skill 路由可视化

每次查询完成后显示：

- 本次使用了哪些 skill
- 为什么使用这些 skill
- 哪个 skill 给出了主要结果

示例：

- `问财选A股`：负责初筛候选池
- `行情数据查询`：补充量价和资金流
- `公告搜索`：补充催化来源

## 11.3 结果区设计

同一轮结果尽量标准化为以下几种视图：

- 概览卡片
- 股票表格
- 比较表
- 风险提示块
- 追问建议区

### 表格最常见字段

- 代码
- 名称
- 所属行业 / 板块
- 核心逻辑
- 最新价 / 涨跌幅
- 关键财务指标
- 事件催化
- 风险点
- 数据来源

## 11.4 追问体验

系统需要自动生成追问建议，例如：

- `把这 5 只按胜率排序`
- `只保留市值 200 亿以上的`
- `去掉最近涨幅过大的`
- `补充最近 7 天公告`
- `对比这 3 只的财务质量`

## 11.5 收藏与候选池

每条结果支持：

- 收藏股票
- 保存到候选池
- 打标签
- 备注

候选池可按：

- 短线
- 波段
- 中线
- 观察
- 放弃

进行分类。

### 12.1 页面一：工作台

核心组件：

- 结构化结果卡

### 12.2 页面二：模板中心

- 管理常用提问模板

核心组件：

### 12.3 页面三：候选池

目标：

- 统一管理感兴趣的股票和会话结果

核心组件：

### 12.4 页面四：设置

核心组件：

## 13. 后端能力需求

产品层面对后端的要求：

1. 能理解问句属于哪种模式。
2. 能把一个问题路由到一个或多个 skills。
3. 能把多 skill 结果整合成统一结构。
4. 能持久化会话、用户画像和收藏结果。
5. 能支持追问继承上下文。
6. 能缓存常见结果，提高响应速度。

## 14. 非功能需求

### 14.1 性能

- 问句提交后立即进入加载态
- 先展示执行中的 skill
- 允许分段返回

### 14.2 稳定性

- 单个 skill 失败不能导致整轮失败
- 需要 fallback 文案
- 查询失败时可重试

### 14.3 可解释性

- 显示本轮 skill 路由
- 区分原始数据和系统总结
- 记录最终使用的查询语句

### 14.4 隐私

- 个人设置默认保存在本地或自有数据库
- 不做不必要的外部上传

## 15. MVP 验收标准

满足以下条件即可视为可用：

1. 你可以不手动挑 skill，直接自然语言发问。
2. 系统能识别短线 / 波段 / 中线价值三类问题。
3. 结果可以形成股票候选表，而不是纯文本长回答。
4. 可以对上一轮结果进行比较、过滤、继续追问。
5. 个人参数可配置且能自动带入。
6. 能查看每轮实际用了哪些 skill。
